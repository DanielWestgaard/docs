Riftmap connects to your GitHub organization or GitLab group and builds a live, queryable dependency graph across every repository — automatically. When you're about to refactor a shared Terraform module, update a base Docker image, or cut a new version of an internal Python package, Riftmap tells you exactly which downstream repositories will be affected before you ship.

## The core problem Riftmap solves

In a multi-repo organization, answering "if I change repo X, what else breaks?" usually means searching through wikis, asking teammates, or discovering breakage after the fact in CI. The more repos you have, the worse it gets.

Riftmap solves this by scanning every repository in your org and resolving how each one depends on every other — across ecosystems, across teams, across hundreds of repos — without asking you to maintain a single configuration file.

## Key features

**Zero per-repo configuration.** Connect once at the org level with a read-only token. Riftmap discovers dependencies from the actual code: `go.mod` files, `requirements.txt`, Dockerfiles, `Chart.yaml`, GitHub Actions workflows, and more.

**Cross-ecosystem dependency graph.** A single scan maps Terraform module references, Docker base images, Python packages, CI/CD pipeline includes, Helm chart dependencies, Go modules, npm packages, Kustomize resources, Kubernetes manifests, and Ansible roles — all in one graph.

**Impact analysis.** Select any repository and run an impact query to see the full downstream blast radius: which repos depend on it, how many hops away, and via what dependency type.

**Interactive graph UI.** Browse your entire org's dependency graph in the browser. Focus Mode narrows the view to a single repository's neighborhood. Impact Mode highlights the downstream blast radius of a change. Semantic zoom, keyboard-driven search, and dependency-type filtering make large graphs navigable.

**REST API.** Query impact programmatically from CI pipelines, internal tooling, or scripts. All graph data is available via a versioned JSON API authenticated with a workspace API key.

## Supported ecosystems

Riftmap parses dependencies from the following ecosystems in a single scan:

Terraform · Docker / docker-compose · Python (requirements.txt, pyproject.toml, setup.cfg) · GitHub Actions · GitLab CI · Ansible (roles + collections) · Helm (Chart.yaml, values.yaml) · Go modules · npm · Kustomize · Kubernetes manifests · ArgoCD
