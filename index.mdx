Riftmap connects to your GitHub organization or GitLab group and automatically discovers how every repository depends on every other — across Terraform modules, Docker images, Python packages, CI/CD pipelines, Helm charts, Go modules, npm packages, and more. No YAML to maintain, no per-repo configuration. Connect once with a read-only token and your dependency graph is live within minutes.

## How Riftmap works

<Steps>
  <Step title="Connect your org">
    Register a GitHub org or GitLab group with a read-only Personal Access Token (or link your OAuth account). No per-repo setup required.
  </Step>
  <Step title="Run a scan">
    Riftmap's workers enumerate every repository in your org, parse dependency declarations across all supported ecosystems, and build a directed dependency graph in seconds.
  </Step>
  <Step title="Explore the graph">
    Browse the interactive graph in the UI, use Focus Mode to zoom into a single repository's neighborhood, or switch to Impact Mode to visualize a change's downstream blast radius.
  </Step>
  <Step title="Query via API">
    Use the REST API from your CI pipelines to check impact before deploying — or to power your own internal tooling.
  </Step>
</Steps>
